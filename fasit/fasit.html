app.py — del 1 FASIT FORKLART

Imports og konfigurasjon

Toppen av app.py — alt vi henter inn og setter opp før vi begynner.

app.py — toppenPython
from flask import (Flask, render_template, redirect,
                    url_for, request, flash, jsonify, abort)
from flask_sqlalchemy import SQLAlchemy
from flask_login import (LoginManager, UserMixin,
                              login_user, logout_user,
                              login_required, current_user)
from werkzeug.security import generate_password_hash, check_password_hash
from functools import wraps
import markdown as md

app = Flask(__name__)
app.config['SECRET_KEY']            = 'bytt-til-noe-hemmelig-og-langt-123!'
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///database.db'

db = SQLAlchemy(app)
Linje for linje
  • from flask import (...) — Vi henter bare de funksjonene vi faktisk trenger fra Flask. render_template sender HTML til nettleseren, redirect sender brukeren til en annen URL, request lar oss lese data brukeren sendte, flash lagrer engangs-meldinger, jsonify sender JSON tilbake til JavaScript, abort avbryter med en HTTP-feilkode.
  • from flask_sqlalchemy import SQLAlchemy — SQLAlchemy er det som lar oss skrive Python-klasser istedenfor SQL-spørringer direkte.
  • from flask_login import (...) — Flask-Login håndterer alt rundt innlogging: hvem er innlogget, beskytt sider, logg inn/ut.
  • from werkzeug.security import generate_password_hash, check_password_hash — Aldri lagre passord i klartekst. Disse to funksjonene krypterer passordet enveiskryptert — du kan aldri dekryptere det tilbake, bare sjekke om et forsøk stemmer.
  • from functools import wraps — Brukes når vi lager vår egen dekorator. wraps sørger for at funksjonens navn og dokumentasjon bevares etter innpakking.
  • import markdown as md — Importerer markdown-biblioteket, men gir det kortnavnet md så vi slipper å skrive markdown.markdown(...) hele tiden.
  • app = Flask(__name__) — Lager Flask-appen. __name__ er Pythons innebygde variabel for "hvilken fil er dette?" — Flask bruker det til å finne templates/ og static/.
  • app.config['SECRET_KEY'] = '...' — En hemmelig streng Flask bruker til å signere session-cookies. Uten denne kan ikke innlogging fungere. Bytt den til noe langt og tilfeldig i et ekte prosjekt.
  • app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///database.db' — Forteller SQLAlchemy at vi bruker SQLite og at filen skal hete database.db. De tre skråstrekene betyr "relativ til prosjektmappen".
  • db = SQLAlchemy(app) — Kobler databasen til Flask-appen. Nå er db tilgjengelig overalt i koden vår.
🧠
Hvorfor import på toppen?

Python kjører filen fra topp til bunn. Alt som trengs av andre ting må importeres før det brukes. Det er konvensjon å samle alle imports øverst — da ser du på ett sted hva prosjektet er avhengig av.

app.py — del 2 FASIT FORKLART

Flask-Login + databasemodeller

Sett opp innloggingssystem og beskriv alle tabeller som Python-klasser.

app.py — login managerPython
login_manager = LoginManager(app)
login_manager.login_view = 'login'

@login_manager.user_loader
def load_user(user_id):
    return User.query.get(int(user_id))
Hva skjer her?
  • LoginManager(app) — Kobler Flask-Login til appen. Nå er current_user tilgjengelig overalt.
  • login_manager.login_view = 'login' — Hvis noen prøver å gå til en beskyttet side uten å være innlogget, send dem til ruten som heter 'login'. Merk: det er funksjonsnavnet, ikke URL-en.
  • @login_manager.user_loader — Flask-Login trenger å vite hvordan den finner en bruker fra en ID. Denne funksjonen kalles automatisk ved hver forespørsel med session-cookiens bruker-ID. Vi henter brukeren fra databasen.
  • int(user_id) — Cookies lagrer alt som tekst. Vi konverterer til heltall fordi databasens ID er et heltall.
Databasemodeller
🧠
Hva er en modell?

En Python-klasse som arver fra db.Model blir automatisk til en databasetabell. Hver klasseattributt med db.Column() blir en kolonne. SQLAlchemy oversetter dette til SQL bak kulissene — du slipper å skrive CREATE TABLE selv.

app.py — alle 6 modellerPython
class User(UserMixin, db.Model):
    id            = db.Column(db.Integer, primary_key=True)
    username      = db.Column(db.String(80),  unique=True,  nullable=False)
    password_hash = db.Column(db.String(200),                  nullable=False)
    is_admin      = db.Column(db.Boolean, default=False)

class Section(db.Model):
    id          = db.Column(db.Integer, primary_key=True)
    title       = db.Column(db.String(100), nullable=False)
    description = db.Column(db.Text)
    emoji       = db.Column(db.String(10))
    topics      = db.relationship('Topic', backref='section', lazy=True,
                                   cascade='all, delete-orphan')

class Topic(db.Model):
    id         = db.Column(db.Integer, primary_key=True)
    title      = db.Column(db.String(100), nullable=False)
    section_id = db.Column(db.Integer, db.ForeignKey('section.id'), nullable=False)
    elements   = db.relationship('LearningElement', backref='topic', lazy=True,
                                   cascade='all, delete-orphan')

class LearningElement(db.Model):
    id             = db.Column(db.Integer, primary_key=True)
    topic_id       = db.Column(db.Integer, db.ForeignKey('topic.id'), nullable=False)
    type           = db.Column(db.String(20),  nullable=False)
    content        = db.Column(db.Text)
    question       = db.Column(db.Text)
    option_a       = db.Column(db.String(200))
    option_b       = db.Column(db.String(200))
    option_c       = db.Column(db.String(200))
    option_d       = db.Column(db.String(200))
    correct_answer = db.Column(db.String(1))
    answer_key     = db.Column(db.Text)

class Progress(db.Model):
    id         = db.Column(db.Integer, primary_key=True)
    user_id    = db.Column(db.Integer, db.ForeignKey('user.id'),             nullable=False)
    element_id = db.Column(db.Integer, db.ForeignKey('learning_element.id'), nullable=False)
    correct    = db.Column(db.Boolean)

class OpenAnswer(db.Model):
    id         = db.Column(db.Integer, primary_key=True)
    user_id    = db.Column(db.Integer, db.ForeignKey('user.id'),             nullable=False)
    element_id = db.Column(db.Integer, db.ForeignKey('learning_element.id'), nullable=False)
    answer     = db.Column(db.Text)

class TopicVisit(db.Model):
    id       = db.Column(db.Integer, primary_key=True)
    user_id  = db.Column(db.Integer, db.ForeignKey('user.id'),  nullable=False)
    topic_id = db.Column(db.Integer, db.ForeignKey('topic.id'), nullable=False)
Nøkkelkonsepter i modellene
  • class User(UserMixin, db.Model) — Arver fra to foreldre. db.Model gjør den til en databasetabell. UserMixin er fra Flask-Login og legger til metodene is_authenticated, is_active og get_id() som Flask-Login forventer å finne.
  • primary_key=True — Denne kolonnen er tabellens unike ID. SQLite teller den automatisk opp for hver ny rad (1, 2, 3...).
  • unique=True — Databasen nekter to rader med samme verdi. Perfekt for brukernavn.
  • nullable=False — Databasen nekter å lagre en rad uten denne verdien. Et krav på databasenivå.
  • db.ForeignKey('section.id') — Denne kolonnen må inneholde en ID som faktisk eksisterer i section-tabellen. Det er slik vi kobler tabeller sammen.
  • db.relationship('Topic', backref='section', cascade='all, delete-orphan') — Definerer en Python-kobling mellom klasser. backref='section' betyr at du kan skrive topic.section for å hente seksjonen. cascade='all, delete-orphan' betyr: slett alle temaer automatisk når seksjonen slettes — ingen foreldreløse rader.
  • db.Column(db.Text) — Brukes for langt innhold uten grense (Markdown-tekst, svar). db.String(100) er for korte strenger med makslengde.
app.py — del 3 FASIT FORKLART

DB-init, admin-dekorator og auth-ruter

Opprett tabeller, admin-bruker, egendefinert tilgangskontroll og innlogging.

app.py — initialiseringPython
with app.app_context():
    db.create_all()
    if not User.query.filter_by(username='admin').first():
        db.session.add(User(
            username='admin',
            password_hash=generate_password_hash('admin123'),
            is_admin=True
        ))
        db.session.commit()
Hva skjer her?
  • with app.app_context(): — Flask bruker en "kontekst" for å vite hvilken app som er aktiv. Utenfor HTTP-forespørsler finnes ikke denne automatisk. Vi oppretter den manuelt for å kunne bruke databasen under oppstart.
  • db.create_all() — Lager alle tabeller som ikke finnes fra før. Trygt å kjøre hver gang — eksisterende tabeller røres ikke.
  • User.query.filter_by(username='admin').first() — Søk etter admin-brukeren. .first() returnerer første treff eller None. Sjekken forhindrer at vi lager to admin-brukere.
  • generate_password_hash('admin123') — Konverterer passordet til en kryptografisk hash. Lagres i databasen. Passordet kan aldri hentes ut igjen.
  • db.session.add() — Legg til i transaksjonen (ikke skrevet til disk ennå).
  • db.session.commit() — Skriv alt i transaksjonen til databasen permanent.
app.py — admin_required dekoratorPython
def admin_required(f):
    @wraps(f)
    @login_required
    def decorated(*args, **kwargs):
        if not current_user.is_admin:
            abort(403)
        return f(*args, **kwargs)
    return decorated
Dekoratoren forklart
  • def admin_required(f): — En dekorator er en funksjon som tar inn en annen funksjon og returnerer en ny, forbedret versjon av den. f er den opprinnelige route-funksjonen.
  • @wraps(f) — Uten dette mister den innpakkede funksjonen sitt opprinnelige navn. Flask bruker funksjonsnavnet til å identifisere ruter internt — @wraps bevarer det.
  • @login_required — Stacker en annen dekorator. Sjekker først at brukeren er innlogget.
  • abort(403) — Avbryter forespørselen med HTTP-statuskode 403 (Forbidden). Nettleseren viser en feilside.
  • return f(*args, **kwargs) — Hvis alle sjekker passerer: kjør den opprinnelige funksjonen med alle argumentene den fikk.
app.py — login, register, logg-utPython
@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        user = User.query.filter_by(username=request.form['username']).first()
        if user and check_password_hash(user.password_hash, request.form['password']):
            login_user(user)
            return redirect(url_for('index'))
        flash('Feil brukernavn eller passord', 'error')
    return render_template('login.html')

@app.route('/register', methods=['GET', 'POST'])
def register():
    if request.method == 'POST':
        username = request.form['username']
        if User.query.filter_by(username=username).first():
            flash('Brukernavnet er allerede tatt', 'error')
            return redirect(url_for('register'))
        db.session.add(User(
            username=username,
            password_hash=generate_password_hash(request.form['password']),
            is_admin=False
        ))
        db.session.commit()
        login_user(User.query.filter_by(username=username).first())
        return redirect(url_for('index'))
    return render_template('register.html')

@app.route('/logg-ut')
@login_required
def logg_ut():
    logout_user()
    return redirect(url_for('login'))
GET vs POST — og hva alt betyr
  • methods=['GET', 'POST'] — En rute svarer bare på GET som standard. Vi legger til POST fordi skjema-submit sender en POST-forespørsel.
  • if request.method == 'POST': — Samme URL brukes til to ting: GET viser skjemaet, POST behandler dataene. Vi sjekker hvilken det er.
  • request.form['username'] — Henter verdien fra HTML-skjemafeltet med name="username". Disse må stemme overens.
  • user and check_password_hash(...) — Python short-circuits: hvis user er None, sjekkes aldri passordet. Forhindrer feil.
  • login_user(user) — Flask-Login lager en session-cookie og husker brukeren.
  • flash('melding', 'error') — Lagrer en engangsmelding som vises én gang i base.html og forsvinner etterpå.
  • url_for('index') — Genererer URL til ruten kalt index. Bruk alltid dette istedenfor hardkodede stringer som '/'.

app.py — innholdsruterPython
@app.route('/')
@login_required
def index():
    sections = Section.query.all()
    return render_template('index.html', sections=sections)

@app.route('/section/<int:section_id>')
@login_required
def section(section_id):
    seksjon = Section.query.get_or_404(section_id)
    return render_template('section.html', section=seksjon)

@app.route('/topic/<int:topic_id>')
@login_required
def topic(topic_id):
    tema = Topic.query.get_or_404(topic_id)

    # Registrer første besøk
    if not TopicVisit.query.filter_by(user_id=current_user.id, topic_id=topic_id).first():
        db.session.add(TopicVisit(user_id=current_user.id, topic_id=topic_id))
        db.session.commit()

    # Sett av hvilke elementer brukeren har besvart
    answered_ids = {p.element_id for p in
                    Progress.query.filter_by(user_id=current_user.id).all()}
    open_answers = {a.element_id: a.answer for a in
                    OpenAnswer.query.filter_by(user_id=current_user.id).all()}

    # Konverter markdown til HTML
    elements_rendered = []
    for el in tema.elements:
        rendered = el.content
        if el.type == 'markdown':
            rendered = md.markdown(el.content or '',
                                   extensions=['fenced_code', 'tables'])
        elements_rendered.append((el, rendered))

    return render_template('topic.html',
        topic=tema,
        elements_rendered=elements_rendered,
        answered_ids=answered_ids,
        open_answers=open_answers)

@app.route('/profil')
@login_required
def profile():
    uid      = current_user.id
    answered = Progress.query.filter_by(user_id=uid).count()
    correct  = Progress.query.filter_by(user_id=uid, correct=True).count()
    visited  = TopicVisit.query.filter_by(user_id=uid).count()
    topic_stats = []
    for tema in Topic.query.all():
        total = len(tema.elements)
        if total == 0: continue
        done = Progress.query.filter_by(user_id=uid).join(LearningElement).filter(
            LearningElement.topic_id == tema.id).count()
        topic_stats.append({
            'topic': tema, 'done': done, 'total': total,
            'percent': int(done / total * 100)
        })
    return render_template('profile.html',
        answered=answered, correct=correct,
        visited=visited, topic_stats=topic_stats)
Viktige teknikker her
  • /section/<int:section_id> — Dynamisk URL. <int:section_id> er en parameter som Flask trekker ut fra URL-en og sender til funksjonen automatisk. Besøker du /section/3 kalles funksjonen med section_id=3.
  • .get_or_404(section_id) — Hent fra databasen, eller returner automatisk en 404-feilside hvis ID-en ikke eksisterer. Mye bedre enn å krasje med en Python-feil.
  • {p.element_id for p in Progress.query...} — Et set comprehension. Lager et Python-sett (ikke en liste) med alle element-IDer brukeren har besvart. Sett er lynraske på oppslag — if id in answered_ids er O(1).
  • {a.element_id: a.answer for a in OpenAnswer.query...} — Et dict comprehension. Lager en ordbok: {element_id: svar}. I malen kan vi da hente svaret med open_answers[element.id].
  • elements_rendered.append((el, rendered)) — Legger til et tuppel (to verdier pakket sammen) i listen. I Jinja2 pakker vi det ut med {% for element, rendered in elements_rendered %}.
  • .join(LearningElement).filter(...) — SQL JOIN: kobler Progress-tabellen til LearningElement-tabellen for å filtrere på topic_id. Uten JOIN måtte vi hente alle rader til Python og filtrere der — mye tregere.
app.py — svar-ruterPython
@app.route('/answer/quiz', methods=['POST'])
@login_required
def answer_quiz():
    data       = request.get_json()
    element_id = data['element_id']
    answer     = data['answer']
    element    = LearningElement.query.get_or_404(element_id)
    if Progress.query.filter_by(user_id=current_user.id, element_id=element_id).first():
        return jsonify({'error': 'allerede besvart'}), 400
    correct = (answer == element.correct_answer)
    db.session.add(Progress(user_id=current_user.id,
                             element_id=element_id, correct=correct))
    db.session.commit()
    return jsonify({'correct': correct, 'right_answer': element.correct_answer})

@app.route('/answer/open/<int:element_id>', methods=['POST'])
@login_required
def answer_open(element_id):
    element = LearningElement.query.get_or_404(element_id)
    if not OpenAnswer.query.filter_by(user_id=current_user.id,
                                        element_id=element_id).first():
        db.session.add(OpenAnswer(user_id=current_user.id,
                                   element_id=element_id,
                                   answer=request.form['answer']))
        db.session.commit()
    return redirect(url_for('topic', topic_id=element.topic_id))
JSON vs skjema — to forskjellige måter å sende data
  • request.get_json() — JavaScript sender JSON (ikke et HTML-skjema). Bruk denne funksjonen istedenfor request.form. Den parser JSON-teksten til en Python-ordbok.
  • jsonify({...}) — Pakk en Python-ordbok som et JSON-svar. JavaScript-koden vår kan da lese dette.
  • return jsonify(...), 400 — Det andre tallet er HTTP-statuskoden. 400 = Bad Request. JavaScript sjekker response.ok — den er false ved 400.
  • correct = (answer == element.correct_answer) — Enkel sammenligning. Resultatet (True/False) lagres i databasen og sendes tilbake til JavaScript.

Mønsteret i CRUD-rutene
  • request.form.get('description', '') — Tryggere enn request.form['description']. .get() returnerer standardverdien (tom streng) hvis feltet mangler — istedenfor å krasje med KeyError.
  • request.form['title'] — Uten .get() — brukes når feltet er obligatorisk (har required i HTML). Krasjer bevisst hvis det mangler.
  • [(User.query.get(p.user_id), p.correct) for p in ...] — Et list comprehension som lager en liste av tuplene: (bruker-objekt, riktig-eller-ikke). I malen loopes over dem med {% for user, value in rows %}.
  • if __name__ == '__main__': app.run(debug=True) — Starter Flask sin innebygde server. debug=True gir automatisk reload ved endringer og feilmeldinger i nettleseren. Fjern dette i produksjon.

Jinja2 i base.html forklart
  • {% block title %}Læringsplattform{% endblock %} — Et navngitt hull. Teksten mellom er standardverdi. Undermaler overskriver den med {% block title %}Min side{% endblock %}.
  • {{ url_for('static', filename='style.css') }} — Doble krøllparenteser = skriv ut en verdi. url_for genererer riktig URL til filen i static/. Bruk alltid dette istedenfor hardkodet sti.
  • {% if current_user.is_authenticated %} — current_user er alltid tilgjengelig i alle maler takket være Flask-Login. Viser innlogget-meny eller gjest-meny basert på dette.
  • {% with messages = get_flashed_messages(with_categories=true) %} — Henter flash-meldinger. with_categories=true betyr vi får tupler (kategori, melding) slik at vi kan style dem forskjellig (error = rød, success = grønn).
  • {% block content %}{% endblock %} — Det viktigste hullet. Alt sideinnhold fra undermaler settes inn her.

document.querySelectorAll('.quiz-btn').forEach(btn => {
  btn.addEventListener('click', async () => {
    const block     = btn.closest('.quiz-block')
    const elementId = parseInt(block.dataset.elementId)
    const answer    = btn.dataset.answer

    const res = await fetch('/answer/quiz', {
      method:  'POST',
      headers: { 'Content-Type': 'application/json' },
      body:    JSON.stringify({ element_id: elementId, answer })
    })
    if (!res.ok) return

    const data = await res.json()
    block.querySelectorAll('.quiz-btn').forEach(b => {
      b.disabled = true
      if (b.dataset.answer === data.right_answer) b.classList.add('correct')
      else if (b === btn && !data.correct)      b.classList.add('wrong')
    })
  })
})
</script>

Nøkkeldetaljer i topic.html
  • {{ rendered | safe }} — Jinja2 escaper HTML som standard (sikkerhetstiltak mot XSS). | safe sier "dette er trygt HTML" og renderer det som ekte HTML istedenfor å vise det som tekst. Bruk kun når du selv genererte HTML-en.
  • data-element-id="{{ element.id }}" — HTML data--attributter er et standardsett for å lagre egendefinerte data i HTML-elementer. JavaScript leser dem med element.dataset.elementId (bindestreker konverteres til camelCase).
  • {% if element.id in answered_ids %}disabled{% endif %} — Betingelse midt inne i et HTML-tag. Legger til disabled-attributtet bare hvis spørsmålet er besvart.
  • async () => { ... } — Asynkron pilfunksjon. async gjør at vi kan bruke await — koden venter på svar fra serveren uten å fryse nettleseren.
  • btn.closest('.quiz-block') — Klatre opp i DOM-treet til nærmeste foreldreelement med klassen. Viktig fordi vi trenger quiz-blokken, ikke bare knappen.
  • JSON.stringify({ element_id: elementId, answer }) — Konverter JavaScript-objekt til JSON-tekst. answer er ES6-forkortelse for answer: answer.
  • b.classList.add('correct') — Legg til CSS-klasse dynamisk. I style.css er .quiz-btn.correct stylet med grønn farge.

  if (localStorage.getItem(key) === 'collapsed') {

  }
  h.addEventListener('click', e => {

    const hidden = body.style.display === 'none'
    body.style.display = hidden ? '' : 'none'
    arrow.textContent  = hidden ? '▼' : '▶'

JavaScript i admin-panelet
  • style="display:none" på alle felt-grupper — Alle tre felt-gruppene (markdown/quiz/open) starter skjult. JavaScript viser riktig gruppe basert på dropdown-valget.
  • document.getElementById('fields-' + this.value) — Siden ID-ene heter fields-markdown, fields-quiz og fields-open, og dropdown-verdiene er 'markdown', 'quiz', 'open', kan vi bygge ID-en dynamisk ved å koble dem.
  • mdInput.addEventListener('input', ...) — 'input'-hendelsen fyres av ved hvert tastetrykk. Raskere feedback enn 'change' som bare fyres når feltet mister fokus.
  • marked.parse(mdInput.value) — Det eksterne biblioteket marked.js konverterer Markdown til HTML direkte i nettleseren — ingen serverrunde nødvendig.
  • if (e.target.closest('form')) return — Slett-knappen ligger inne i header-elementet som lytter på klikk for kollaps. Uten denne linjen ville klikk på slett-knappen også kollapse seksjonen.
  • localStorage.setItem(key, ...) — Lagrer kollaps-tilstanden i nettleseren. Neste gang siden lastes gjenopprettes tilstanden — admin slipper å åpne alle seksjonene på nytt.

De viktigste CSS-teknikkene forklart
  • :root { --primary: #4f46e5; } — CSS custom properties (variabler). Definer farger og verdier én gang øverst. Endre én linje → oppdateres overalt i filen.
  • position: sticky; top: 0; z-index: 100; — Navigasjonen "klistrer seg" til toppen av vinduet når du scroller. z-index: 100 gjør at den legger seg over annet innhold.
  • grid-template-columns: repeat(auto-fill, minmax(230px, 1fr)) — Responsivt rutenett uten media queries. "Lag flest mulig kolonner, ingen smalere enn 230px." Fungerer på alle skjermstørrelser automatisk.
  • margin-right: auto på .nav-logo — I flexbox skyver dette logoen til venstre og alt annet til høyre, uten å spesifisere bredder.
  • transition: transform .15s, box-shadow .15s — Animer flere egenskaper på én gang. Gjør hover-effekten glatt.
  • box-shadow: 0 0 0 3px rgba(79,70,229,.12) på focus — En synlig ring rundt fokusert skjemafelt (tilgjengelighet). x=0, y=0, blur=0, spread=3px. Bruker spread istedenfor blur for skarp, tydelig ring.
  • .quiz-btn:hover:not(:disabled) — CSS-selektor med negasjon: hover-stil gjelder bare hvis knappen ikke er disabled. Forhindrer at deaktiverte knapper ser ut som de er klikkbare.
  • white-space: nowrap; overflow: hidden; text-overflow: ellipsis; — Tre CSS-egenskaper som til sammen gir "..." på tekst som er for lang for plassen.
  • @media (max-width: 600px) — Media query. Alt inni gjelder bare på skjermer smalere enn 600px (mobil). Kjerne i responsiv design.
